
Create backup of production database manually:
docker run --rm --volumes-from komidabot-docker_komidabot-db_1 -v $(pwd):/backup ubuntu tar czf /backup/backup.tgz /var/lib/postgresql/data

Restore the backup:
docker run --rm --volumes-from komidabot-docker_komidabot-db_1 -v $(pwd):/backup ubuntu bash -c "cd /var/lib/postgresql/data && tar xzf /backup/backup.tgz --strip 4"

Or apply it to the dev bot:
docker run --rm --volumes-from komidabot-docker_komidabot-dev-db_1 -v $(pwd):/backup ubuntu bash -c "cd /var/lib/postgresql/data && tar xzf /backup/backup.tgz --strip 4"
